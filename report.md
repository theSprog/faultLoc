抹去 safe value 对 FIC 的影响

- 有可能定位不到错误
  - 错误: (1,-,-,-) 和 (-,1,-,-)
  - (0,1,1,1) -> 定位不出,因为在对第1个进行 mutate 时引入了新故障
  - 从而 pending 也计算不准确了
- 有可能定位到错误的重叠部分
  - 错误: (1,1,-,-) 和 (1,-,1,-)
  - (1,1,1,1) -> 定位到重叠

-----

即使有非单因子故障,FIC 也可能 pending 为 0

i.e.
错误: (1,1,-,-)
测试用例: (1,1,1,1)

通过: (1,-,1,1) 和 (-,1,1,1)

MinFault: (1,1,-,-)

pendingSize = 0
